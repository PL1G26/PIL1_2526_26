Workflow Git

Le groupe a adopté un workflow à trois niveaux de branches pour éviter les conflits et garder le code main toujours stable :

main       ← code stable, évalué en fin de projet
├── develop    ← branche d'intégration commune
├── frontend
└── backend

Chaque développeur travaillait sur sa propre branche et ouvrait une Pull Request vers develop à la fin de chaque fonctionnalité. Les commits suivaient les conventions : feat:, fix:, docs:, style:, test:.

04 — Base de données

Schéma de la base de données

La base de données PostgreSQL (hébergée sur Supabase) contient 9 tables. La table users est centrale ; toutes les autres en dépendent directement ou indirectement.

users

first_name, last_name VARCHAR
email VARCHAR UNIQUE
phone_number VARCHAR UNIQUE
password_hash VARCHAR
field_of_study, level, bio
skills
PK id SERIAL

created_at TIMESTAMP DEFAULT NOW()
user_skills
PKFK user_id → users
PKFK skill_id → skills
proficiency 'strong'|'weak'
user_availabilities
PK id SERIAL
FK user_id → users
day_of_week VARCHAR
start_time, end_time TIME
mentorship_posts
PK id SERIAL
FK user_id → users
type 'offer'|'request'
FK skill_id → skills
mode 'online'|'offline'|'both'
is_active BOOLEAN
post_availabilities
PK id SERIAL
FK post_id → mentorship_posts
day_of_week, start_time, end_time
matches
PK id SERIAL
FK mentor_id → users
FK mentee_id → users
FK skill_id → skills
score DECIMAL(5,2)
status 'pending'|'accepted'|'rejected'
conversations
PK id SERIAL
FK match_id → matches (UNIQUE)
created_at, updated_at TIMESTAMP
messages
PK id SERIAL

content TEXT

Relations principales

users ──(1:N)→ user_skills ←(1:N)── skills
users ──(1:N)→ user_availabilities
users ──(1:N)→ mentorship_posts ──(1:N)→ post_availabilities
users(mentor) ──(1:N)→ matches ←(1:N)── users(mentee)
matches ──(1:1)→ conversations ──(1:N)→ messages

Algorithme de matching — Score de compatibilité

Le cœur de la plateforme. Il calcule un score de compatibilité sur 100 points pour chaque paire mentor-mentoré selon 3 critères cumulables. Seuls les matches avec un score ≥ 40 points sont proposés, triés par score décroissant.

40

Compétences

Le mentor a la matière en strong, le mentoré l'a en weak → correspondance parfaite.

30

Disponibilités

+10 pts par créneau horaire commun entre mentor et mentoré, plafonné à 30 pts.

30

Filière / Niveau

Même filière → +30 pts. Filières proches (ex. IA/IM) → +15 pts. Différentes → +0 pt.

Compatibilité des compétences 40 pts
Disponibilités communes (max 3 créneaux) 30 pts
Proximité de filière 30 pts

Documentation de l'API

L'API REST FastAPI expose tous ses endpoints via Swagger UI sur /docs. Les routes privées requièrent un header Authorization: Bearer <token>.

Authentification

POST/api/auth/registerInscription d'un nouvel utilisateur
POST/api/auth/loginConnexion → retourne un token JWT
GET/api/auth/me🔒Profil de l'utilisateur connecté

Utilisateurs & Compétences

PUT/api/users/me🔒Modifier son profil
PUT/api/users/me/skills🔒Mettre à jour les compétences (strong/weak)
PUT/api/users/me/availabilities🔒Configurer les créneaux de disponibilité
GET/api/skillsListe de toutes les compétences disponibles

Posts de mentorat

GET/api/postsTous les posts actifs (filtrable)
POST/api/posts🔒Créer une offre ou demande de mentorat
PUT/api/posts/{id}🔒Modifier un post

Matching & Messagerie

GET/api/matches🔒Matches calculés par l'algorithme
PUT/api/matches/{id}/accept🔒Accepter → crée une conversation
PUT/api/matches/{id}/reject🔒Refuser un match
GET/api/conversations🔒Conversations avec messages non-lus
POST/api/conversations/{id}/messages🔒Envoyer un message

Étape 1 — Cloner le dépôt

git clone https://github.com/PL1G26/PIL1_2526_26.git
cd PIL1_2526_26

Étape 2 — Initialiser la base de données

Option A — Supabase (recommandé) : créer un projet sur supabase.com, coller le contenu de backend/schema.sql dans l'éditeur SQL, puis récupérer l'URL et les clés.

# Option B — PostgreSQL local
psql -U postgres -c "CREATE DATABASE ifri_mentorlink;"
psql -U postgres -d ifri_mentorlink -f backend/schema.sql

Étape 3 — Configurer et lancer le backend

cd backend

# Créer et activer l'environnement virtuel
python -m venv .venv
source .venv/bin/activate        # Linux / macOS
# .venv\Scripts\activate         # Windows

# Installer les dépendances
pip install -r requirements.txt

# Configurer les variables d'environnement
cp .env.example .env
# → Éditer .env : DATABASE_URL, SECRET_KEY, SUPABASE_URL, SUPABASE_ANON_KEY

# Lancer le serveur de développement
uvicorn main:app --reload --host 0.0.0.0 --port 8000
# API : http://localhost:8000
# Swagger UI : http://localhost:8000/docs

Étape 4 — Lancer le frontend

cd frontend
npm install
cp .env.example .env
# → Éditer .env : VITE_API_BASE_URL, VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY
npm run dev
# Interface : http://localhost:5173

Étape 5 — Build de production

# Frontend : génère les assets statiques dans dist/
cd frontend && npm run build

# Backend : lancer avec Gunicorn (4 workers)
cd backend
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000

07 — Manuel utilisateur

Manuel d'utilisation

Ce manuel décrit le parcours complet d'un étudiant sur IFRI_MentorLink, de l'inscription jusqu'à la messagerie en temps réel.

1

Inscription

Rendez-vous sur la page d'accueil et cliquez sur S'inscrire. Renseignez vos informations : nom, prénom, e-mail, numéro de téléphone (+229...), mot de passe, filière et niveau. Indiquez vos compétences maîtrisées (points forts) et les matières où vous avez besoin d'aide (points faibles), ainsi que vos disponibilités hebdomadaires. L'e-mail et le numéro doivent être uniques — un token JWT est généré automatiquement.

2

Connexion

Connectez-vous avec votre adresse e-mail (ou numéro de téléphone) et votre mot de passe. Après connexion, vous êtes redirigé vers votre tableau de bord — le token JWT est persisté et maintient la session après rechargement de page.

3

Gérer son profil

Depuis l'onglet Profil du dashboard, modifiez vos informations (bio, filière, niveau, photo), ajoutez ou retirez des compétences — Point fort (je peux enseigner) ou Point faible (j'ai besoin d'aide) — et configurez vos disponibilités horaires : jour, heure de début et de fin.

4

Publier une offre ou une demande de mentorat

Dans l'onglet Mes Posts, cliquez sur Créer un post. Choisissez le type (Offre si vous pouvez aider, Demande si vous avez besoin d'aide), la matière concernée, le format (présentiel / en ligne / les deux) et vos créneaux. Un utilisateur peut avoir plusieurs posts actifs simultanément.

5

Explorer les offres et demandes

L'onglet Rechercher affiche tous les posts actifs de la communauté avec filtres par matière, type ou format. Vous pouvez répondre directement à une offre ou demande qui vous correspond.

6

Consulter les correspondances (matching)

L'onglet Correspondances affiche les paires mentor-mentoré calculées automatiquement. Chaque carte indique le nom, les compétences communes, les créneaux partagés et un score sur 100. Cliquez sur Accepter pour confirmer la mise en relation ou Refuser pour la rejeter.

7

Messagerie instantanée

Une fois un match accepté, une conversation est créée automatiquement. Accédez à l'onglet Messages pour discuter en temps réel — les messages arrivent instantanément via Supabase Realtime. L'historique complet est conservé. Seules les conversations issues de matches acceptés y figurent.
